 			Autosomes (Pairs 1–11)
These carry instructions for general body development and functions[affects Max lvl treshold]. 
Chromosome 1: Power = Strength + Dexterity = How much force the character generates
Chromosome 2: Accuracy = Perception + Dexterity = Charcters hand eye coordination skill
Chromosome 3: Charisma = Perception + Intellingence = Characters social awareness 
Chromosome 4: Stamina = Strength + Endurance = The amount of energy a character can exert at once 
Chromosome 5: Toughness = Willpower + Strength = How much damage a characters body can absorb
Chromosome 6: Quickness = Dexterity + Endurance = How well the character can navigate spacial movement
Chromosome 7: Resilience = Willpower + Percpetion = How well the character can guard against mental attacks &/or recover from them.
Chromosome 8: Muscle Memory = (Intelligence + Endurance + Dexterity)/1.5 = Governs skill retention and potential 
Chromosome 9: Aptitude = (Intelligence + Willpower + Perception)/1.5= How well the character can learn a skill, execute, and or strategize 
Chromosome 10: Constitution/Lifeforce [Qi/Ki/Chakra] = Willpower + Endurance = Represents the characters pysche and body (soul).
Chromosome 11: Metabolism - fat and Muscle threshold
Chromosome 23: Sex Chromosome

Chromosome 18: Resilience = Can Slot [Accuracy, Charisma, Toughness, Resilience, Aptitude, Constitution]
Chromosome 19: Muscle Memory = Can Slot [Accuracy, Charisma, Quickness, Resilience, Muscle-Memory, Aptitude, Constitution]
Chromosome 20: Aptitude = Can Slot [Accuracy, Charisma, Toughness, Resilience, Muscle-Memory, Aptitude, Constitution]

Population model:

The Base Formula The fundamental principle of population change is a simple bookkeeping equation: 
\(N_{t+1}=N_{t}+B-D+I-E\)
Where: \(N_{t+1}\) = Population size in the next time step.
\(N_{t}\) = Current population size.\
(B\) = Births (Natality).
\(D\) = Deaths (Mortality).
\(I\) = Immigration.
\(E\) = Emigration. 

1. Defining Carrying Capacity (\(K\))
# K = Carrying Capacity (Max population the area can sustain)
# Area: Physical space available (e.g., in tiles, acres)
# Resource Availability: (Avg Caloric Value of vegetation - Avg Caloric Need of Species - [Avg Need of Competitors])
# Climate Suitability: Modifier (0.0 to 1.0) based on Climate_Preference and Season_Tolerance

K = Area * Resource_Availability * Climate_Suitability


2. Modeling Population Growth (Births and Deaths)
# Delta N = Change in Population per turn
# r = Max intrinsic growth rate (base value modified by Offspring_Yield genotype)
# Nt = Current population size

Delta_N = r * Nt * (1 - (Nt / K))

3. Refining Mortality (Death Rate)
# D = Total Deaths per turn
# Base_Death_Rate: General mortality rate
# Genetic_Condition_Modifier: Modifier applied for Disadvantages like 'Trisomy 18' (Fated to Die), 'Developmental Failure' risks, or general disease susceptibility.
# Competition_Modifier: Increased death rate based on the density of competitors in the region.

D = (Base_Death_Rate + Genetic_Condition_Modifier + Competition_Modifier) * Nt

 Refining Natality (Birth Rate)
# B = Total Births per turn
# Base_Birth_Rate: General natality rate
# Offspring_Yield_Multiplier: Modifier from the 'High', 'Medium', or 'Low' Offspring_Yield genotype.
# Fertility_Status: 1.0 if fertile, 0.0 if sterile (e.g., odd ploidy levels like 3n or 5n, or specific divergence levels).

B = (Base_Birth_Rate * Offspring_Yield_Multiplier) * Fertility_Status * Nt

Ploidy = Number of chromosome sets

PloidyType: 
   - Haploid (1n)   = Reproductive cells only, non-viable as organism
   - Diploid (2n)   = Normal, baseline
   - Triploid (3n)  = Sterile (odd, can't divide evenly)
   - Tetraploid (4n)= Fertile with 4n, sterile with 2n
   - Pentaploid (5n)= Sterile
   - Hexaploid (6n) = Fertile with 6n, reduced with 3n
   - Higher         = Increasingly unstable

PloidyTypeSize:  S = 1 + 0.72 * ln(n / 2) [average 4n for k]
 
TypeSizeTheorecticalFormula:
S = 1 + k * ln(n / 2)
S: The resulting size ratio (relative to the diploid 2n = 1.0).
1: The base value representing the 100% size of a diploid.
k: The scaling constant (determines how aggressively the size increases).
ln: The Natural Logarithm (log base e).
n: The ploidy level (e.g., 1, 2, 3, 4...).
n / 2: Normalizes the input so that when n=2, the result inside the log is 1, making the size exactly 1.0.

	DIPLOIDIZATION 

Stable Species Threshold: n ≤ 25 pairs (diploid)

Diploidization Rate: 
   - Per generation: 0.1% chance of chromosome fusion
   - Accelerated by:  Inbreeding, environmental stability
   - Resisted by: Outbreeding, environmental variability

High Chromosome Count Risks:
   - n > 50: +10% mutation rate (harmful)
   - n > 75: +20% developmental failure

				Count_A - Count_B| / Max(Count_A, Count_B) × 0.5

			Total Divergence = (Enum Diff + Magnitude Diff + Chromosome Diff) / Max Possible
					Divergence Thresholds:  

				0. 00 - 0.15: Same Species (Variant)
				0.16 - 0.35:  Subspecies
				0.36 - 0.55: Related Species (Cross-fertile)
				0.56 - 0.75: Distant Species (Hybrid possible, fertility risk)
				0.76 - 1.00: Separate Species (Incompatible)

New Species Establishment:  

Requirements:
   1. Divergence Score ≥ 0.36 from origin
   2. Stable traits for 3 generations
   3. Population threshold (breeding pool exists)

On Establishment:
   Species_id: generated
   Origin: recorded (parent species)
   [Trait configuration becomes new baseline]
   [Magnitude ranges reset to current averages]

						STERILITY


Type A: F1 Sterile (Permanent)
   - Occurs when:  Divergence 0.65 - 0.75 OR odd ploidy


Type B: F1 Sterile, F2+ Fertile (Recoverable)
   - Occurs when: Divergence 0.56 - 0.64

Type C:  Reduced Fertility (Degrading)
   - Occurs when:  Divergence 0.45 - 0.55

1. Individual-Food Scoring (The Internal Resolve)Before a meal is combined, each food item is broken down into its Flavor Profile.
Input: (Food1-PrimaryFlavor(Concentration) + SecondaryFlavor(Concentration))
Rule: Primary and Secondary flavors must resolve their internal interaction based on their Type and Concentration.
Concentration States:
Low (0-2): [Bland, Weak, Mild] = Positive
Medium (3-4): [Medium, Strong] = Neutral
High (5-7): [Rich, Full, Sharp] = Negative

2. Meal Scoring (The Conflict & Balance Gate)
When multiple foods are combined, their resolved flavors hit the Interaction Matrix. 
Flavors do not cancel out; they Balance.

Polarity Mappings
Each flavor type carries a weight that determines the Enjoyability Rating.
Primary Polarity:Positive: Sweet
Neutral: Sour, Salty, Umami
Negative: Bitter
Secondary Polarity:
Positive: Mint, Fruity, Berry, Smokey, Tart, Nutty, Floral, Fatty.
Neutral: Melon, Citrus, Fishy, Peppery, Licorice, Piney, Herbal, Starchy.
Negative: Earthy, Briny, Gamey, Acidic.Variable: Spicy, Pungent (Subject to Amount & Tolerance).

3. Texture Profile 
Positive (+1): Tender, Chewy, Moist, Crisp, Delicate.
Neutral (0): Soft, Firm, Juicy, Watery, Creamy, Spongy.
Negative (-1): Mushy, Tough, Fibrous, Waxy, Dry.Modifier (x2): 
Crunchy doubles the intensity of the most prevalent texture type in the meal.

4. The Final Synthesis (The Formula)

The final result is not just a number; it is a Consolidated Profile (Prime, Sec, Neutral, Texture) that is read against the Enjoyability Chart.
The Math Sequence:
Primary Block (P): (Primary1 + Primary2) -> Apply Interaction Gate -> Total P-Intensity.
Secondary Block (S): (Secondary1 + Secondary2) -> Total S-Intensity.
Neutral Block (N): (Neutral_1 x Neutral_2) product.
Texture Block (T): Sum of (Polarity x Intensity) + Modifiers.
The Formula:(P + S) x (N) = Raw Score

Final Output Requirement:
Raw Score: (Integer)
Flavor Balance: (Net Polarity of P and S)
Texture Balance: (Net Polarity of T)
Enjoyability Rating: (The "Final Rate" based on the consolidated profile).


Core Fire Simulation Formulas (For Coooking)
These formulas determine the time required for an item to reach a desired temperature or complete a phase change (like boiling water out of food):
Formula Name	Formula	Notes
1. Fuel Energy Output	Fo = Fm (Fuel Mass[kg]) * Fh (Total Specific Heat of Fuel[kj])	Calculates the total available energy from the fuel source.
2. Element Heat Absorption	Ea = Fo * [Eh (Specific Heat of Element) / Fh]	Calculates how much energy a specific element (e.g., Water in food) absorbs.
3. Time until Phase Change	Tp = [ Et (Energy Threshold/Latent Heat) * Em (Mass of Element)] / Ea	Calculates turns required to complete a change (e.g., boiling all water into steam).
